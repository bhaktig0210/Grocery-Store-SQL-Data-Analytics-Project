# Grocery-Store-SQL-Data-Analytics-Project

🛒 Grocery Store SQL Data Analytics Project

📌 Project Overview

This project focuses on designing and analyzing a relational database for a grocery store using SQL. The database stores information about customers, products, suppliers, employees, orders, and order details. SQL queries are used to analyze the data and generate meaningful business insights that can support data-driven decision-making.

The project demonstrates fundamental database management, SQL querying, and data analysis skills by solving real-world business problems.

🎯 Objectives

* Design a relational database for a grocery store.
* Store and manage transactional data efficiently.
* Retrieve and analyze data using SQL queries.
* Generate business insights from sales and customer data.
* Practice SQL concepts such as joins, aggregations, filtering, and grouping.

📂 Dataset Information

The database consists of the following tables:

* **Customers** – Customer information
* **Orders** – Order details
* **OrderDetails** – Products purchased in each order
* **Products** – Product information
* **Categories** – Product categories
* **Suppliers** – Supplier details
* **Store_Employees** – Employee information

The dataset simulates real-world grocery store operations and relationships using primary and foreign keys.

🔄 Project Workflow

1. Design the relational database schema.
2. Create tables with appropriate primary and foreign keys.
3. Insert and organize grocery store data.
4. Write SQL queries to answer business questions.
5. Analyze query results.
6. Present findings and business insights.

❓ Key Business Questions

* How many unique customers have placed orders?
* How many total orders have been placed?
* How many suppliers are available in the database?
* How many products exist in each category?
* What is the average quantity ordered per product?
* What is the relationship between quantity ordered and total price?
* What is the average product price by supplier?
* What is the average product price by category?
* How many employees have processed customer orders?
* What is the average value of each order?


## 📊 Key Insights

* Customer ordering patterns can be analyzed using transactional data.
* Product categories can be compared based on pricing.
* Supplier-wise pricing helps evaluate supplier performance.
* Average order values provide insights into customer purchasing behavior.
* Employee order processing data helps understand operational efficiency.
* SQL queries enable quick extraction of valuable business information from large datasets.
